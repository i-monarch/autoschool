# UI/UX Guidelines

> Этот файл — справочник. Основные правила в CLAUDE.md.

---

## Принципы
- Целевая аудитория — 16-55 лет, не IT. Интерфейс понятен без инструкции.
- Минимум кликов: регистрация → оплата → первый урок — макс 4-5 шагов.

## DaisyUI тема
- Основная: `light`, поддержка `dark` через toggle
- `primary` — CTA кнопки, `success` — завершено, `warning` — внимание, `error` — ошибки
- Шрифт: системный стек (`font-sans`)
- `rounded-lg` карточки, `shadow-md` контент, `shadow-lg` модалки

## Подсказки и onboarding
- Tooltip на неочевидных кнопках/иконках
- Первый вход — пошаговый onboarding (3-4 шага)
- Empty states с действием ("Ви ще не придбали курс. Оберiть тариф ->")
- Inline validation — ошибки под полем сразу при вводе

## Feedback на действия
- Кнопки: `loading` состояние, блокировка от повторного клика
- Успех: toast зеленый, автоскрытие 3 сек
- Ошибка: toast красный, закрывается вручную
- Skeleton при загрузке, модалка для опасных действий

## Навигация
- Sidebar (desktop) + Bottom tab bar (mobile) — разделы всегда видны
- Breadcrumbs на внутренних страницах
- Hamburger НЕ использовать для основной навигации

## Адаптивность (mobile-first)

Все компоненты сначала под мобильный (320-428px), потом расширяются.

**Breakpoints:** default → `sm:` 640 → `md:` 768 → `lg:` 1024 → `xl:` 1280

**Видео-плеер:**
- Мобильный: полная ширина, fullscreen, landscape
- Кнопки минимум 48x48px, свайп для перемотки
- Автовыбор качества по скорости (360p/720p)
- Запоминание позиции, Picture-in-Picture

**Тесты:**
- Мобильный: 1 вопрос на экран, крупные карточки ответов
- Таймер sticky top-0
- Навигация: горизонтальный скролл (зеленый — отвечено, серый — нет)
- Десктоп: sidebar с навигацией справа

**Формы:**
- Инпуты: высота 48px, шрифт 16px (против автозума iOS)
- Submit на мобильном: `w-full`, фиксирован снизу при длинных формах
- `inputmode="tel"` для телефона, `inputmode="email"` для email

**Карточки:** 1 колонка моб → 2 планшет → 3-4 десктоп

**Текст:** базовый 16px, минимум 14px, `leading-relaxed` для длинных текстов, `max-w-prose`

## PWA / Offline
- Кеширование тестов и результатов offline
- Background Sync для ответов при потере связи
- Индикатор "Немає з'єднання", не ломать UI

## Доступность (a11y)
- `aria-label` на кнопках-иконках
- Контраст минимум 4.5:1 (WCAG AA)
- `focus-visible` стили
- `alt` на изображениях, `<label>` + `htmlFor` для инпутов

## Компоненты — 4 состояния
1. Loading — skeleton/спиннер
2. Success — данные
3. Empty — подсказка + CTA
4. Error — сообщение + "Спробувати знову"
